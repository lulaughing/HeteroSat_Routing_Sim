# H-IGA 参数相融实验描述

## 1. 实验目的

为进一步说明 H-IGA 域内路由机制中拥塞惩罚参数的作用，补充开展了 `γ` 与 `λ` 的参数相融实验。该实验不再比较 H-IGA、SGA 和 Dijkstra 三种算法的总体性能，而是聚焦于 H-IGA 内部拥塞惩罚项的参数组合如何影响有效吞吐量、平均时延与平均丢包率。

本实验的目标有三点：

1. 验证 `γ` 与 `λ` 的变化是否会对 H-IGA 的性能产生可观影响。
2. 判断这种影响是否随网络负载变化而变化。
3. 为论文中默认参数 `γ=2.0`、`λ=1.0` 的选取提供补充解释。

## 2. 实验设计

### 2.1 参数范围

仅调整域内 H-IGA 适应度函数中的拥塞惩罚参数：

- 拥塞敏感度 `γ ∈ {0.5, 1.0, 2.0, 3.0}`
- 拥塞权重 `λ ∈ {0.5, 1.0, 1.5, 2.0}`

域间路由的拥塞参数保持不变，不纳入本次实验，以确保实验结论只反映域内 H-IGA 的参数作用。

### 2.2 负载设置

选取中高负载区间：

- `Load = 300`
- `Load = 400`
- `Load = 500`
- `Load = 600`

选择该区间的原因是，低负载阶段网络整体拥塞较弱，参数差异往往难以显现；而中高负载阶段更能体现拥塞惩罚机制对路由选择的影响。

### 2.3 评价指标

采用以下三个指标：

- 平均有效吞吐量 `AvgGoodput`
- 平均端到端时延 `AvgDelay`
- 平均丢包率 `AvgLoss`

其中，柱状图采用相对于默认参数组合 `γ=2.0, λ=1.0` 的增量形式展示，以便观察不同参数组合带来的细微差异。

## 3. 实验结果概述

### 3.1 低负载端的影响较弱

在 `Load=300` 时，各参数组合之间的性能差异非常有限。例如，`AvgGoodput` 的总体波动范围仅约 `0.23 Mbps`，说明在网络尚未明显拥塞时，H-IGA 对拥塞惩罚参数并不敏感。该结果表明，算法在中低负载阶段具有一定的参数鲁棒性。

### 3.2 中高负载下参数差异开始显现

在 `Load=400` 时，参数组合之间已经出现可观差别，但整体仍以小幅波动为主。最优吞吐组合为 `γ=1.0, λ=0.5`，相较默认参数：

- `AvgGoodput` 提升约 `0.223 Mbps`
- `AvgDelay` 降低约 `0.623 ms`
- `AvgLoss` 降低约 `0.002` 个百分点

这一阶段说明适度降低拥塞惩罚强度有助于提升路径效率，但总体收益尚不显著。

### 3.3 高负载下存在更明显的最优区间

在 `Load=500` 时，参数差异明显扩大。表现最优的组合为 `γ=0.5, λ=1.5`，相较默认参数 `γ=2.0, λ=1.0`：

- `AvgGoodput` 提升约 `35.15 Mbps`
- `AvgDelay` 降低约 `0.90 ms`
- `AvgLoss` 降低约 `0.112` 个百分点

该结果说明，在高负载但尚未进入最极端拥塞状态时，较小的 `γ` 可以避免算法对局部拥塞过度敏感，从而减少不必要的绕路；而适中的 `λ` 又能保留足够的拥塞规避能力，因此更利于取得吞吐量、时延与丢包之间的平衡。

### 3.4 极高负载下需要适度增强惩罚强度

在 `Load=600` 时，最优吞吐组合转向 `γ=2.0, λ=1.5`。相较默认参数：

- `AvgGoodput` 提升约 `34.29 Mbps`
- `AvgLoss` 降低约 `0.0875` 个百分点
- `AvgDelay` 仅增加约 `0.27 ms`

这说明当网络进入更高拥塞状态后，适当提高 `λ` 的取值能够增强算法对热点链路的规避能力，从而降低丢包并提高有效吞吐量；同时，由于 `γ` 保持在 `2.0` 的中等水平，并未导致时延出现明显恶化。

## 4. 结论归纳

从整体趋势来看，本次参数相融实验主要能够说明以下几点：

1. `γ` 与 `λ` 的作用具有明显的负载依赖性。
2. 在中低负载场景下，参数变化对 H-IGA 性能的影响较弱。
3. 在高负载场景下，参数组合会显著影响 H-IGA 的吞吐量与丢包性能。
4. `γ` 更像决定算法“对拥塞变化有多敏感”的主导参数，而 `λ` 更像控制惩罚强度幅值的调节参数。
5. 不存在一组在所有负载下都严格最优的全局参数。

因此，这组实验更适合支持如下论断：

- H-IGA 的拥塞惩罚参数应根据网络负载水平进行折中选择。
- 当前论文中采用的默认参数 `γ=2.0`、`λ=1.0` 并非在所有场景下的绝对最优值，但其在不同负载条件下表现较为均衡，适合作为统一实验配置。
- 若面向更高负载场景进行进一步优化，可考虑适当提高 `λ`，以增强对热点链路的规避能力。

## 5. 论文写作建议

### 5.1 推荐写法

这组实验不建议写成“最优参数搜索”，而应写成“机制验证”。更合适的表达方式是：

- 说明 H-IGA 的拥塞惩罚参数并非静态无关变量，而会影响算法在不同负载下的避拥塞行为。
- 强调该实验用于验证参数变化对算法行为的影响规律，而不是为了给出一个全场景唯一最优值。

### 5.2 不建议过度放大的内容

以下内容不宜写得过强：

- 不宜声称 `γ=0.5, λ=1.5` 或 `γ=2.0, λ=1.5` 是“全局最优参数”。
- 不宜把 `Load=300` 的微小差异写成显著性能提升。
- 不宜围绕个别异常点作过度解释。

### 5.3 建议在正文中的定位

该实验建议放在第四章仿真结果分析部分，在“三算法总体性能对比”和“H-IGA 业务分级 QoS 分析”之后，作为一个补充小节。逻辑顺序可以是：

1. 先证明 H-IGA 相对基线算法有效。
2. 再说明 H-IGA 内部拥塞惩罚参数对高负载性能存在影响。
3. 最后说明默认参数是折中配置，而非任意选取。

## 6. 可直接用于论文的正文草稿

可在第四章中增加如下表述：

> 为进一步分析 H-IGA 域内拥塞惩罚机制中参数取值对算法性能的影响，本文补充开展了拥塞敏感度 `γ` 与拥塞权重 `λ` 的参数相融实验。在该实验中，仅调整域内 H-IGA 适应度函数中的拥塞惩罚参数，域间路由模型保持不变，以保证实验结果能够准确反映 H-IGA 局部避拥塞机制的作用规律。实验负载选取 `300`、`400`、`500` 和 `600` 四个代表性并发业务规模，并从平均有效吞吐量、平均端到端时延和平均丢包率三个维度进行评价。

> 结果表明，在 `Load=300` 的中低负载条件下，不同参数组合之间的性能差异较小，说明 H-IGA 对拥塞惩罚参数具有一定鲁棒性；而在 `Load=500` 和 `Load=600` 的高负载条件下，参数作用开始明显增强。其中，在 `Load=500` 时，较小的拥塞敏感度配合中等强度的惩罚权重能够获得更优的吞吐与时延平衡；在 `Load=600` 时，适当提高惩罚权重则更有利于降低丢包并提升有效吞吐。该结果说明，H-IGA 的拥塞惩罚参数具有一定的负载依赖性，不存在适用于所有场景的唯一最优组合。综合不同负载条件下的整体表现，本文采用 `γ=2.0`、`λ=1.0` 作为统一实验配置，以兼顾算法的稳定性与普适性。
